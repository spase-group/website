Naming Authorities

The naming authority of a resource is generally the entity, agency, or organization that commissioned the creation of the resource. Very often, though not always, the entity that sponsored the creation of the resource, i.e., its funding agency, serves also as the naming authority of the resource. A naming authority is therefore the entity to which a resource described by a SPASE document (having a SPASE Resource ID and a DOI reference) is attributed. For this reason, naming authorities can be used conveniently to organize SPASE metadata.

In support of resource sharing, SPASE resource descriptions should be made publicly available and shared from a registry through either a repository or a service. The preferred method is a publicly accessible git repository. Convenient git hosting services include GitHub and (for self-hosting) GitLab. Each naming authority should correspond to a "Repository" of resources. The SPASE ID for this Repository must be "spase://{NA}/Repository/metadata" where {NA} is replaced with the naming authority name. For example, the NASA repository will have a Repository resource description with the SPASE Resource ID of:

spase://NASA/Repository/metadata

Rules for Assigning a Naming Authority

The naming authority of a resource should be assigned at the highest, most appropriate level of entity to which the commissioning or origination of the resource can be attributed. For example, it could be at the national agency level like it is for the "NASA" naming authority. Or in the case of a conglomerate entity, it could be at the highest project level like it is for "IUGONET". The SPASE Group has devised an order of priority when assigning a naming authority, as follows:

  1. Major oversight/funding agency (CNES, ESA, JAXA, NASA, NOAA, NSF,…etc.) for resources from missions, observatories, instruments or investigations that fall well within an agency management;
  2. Overall long-term project name (e.g., ISWI, GONG, IUGONET, Nançay,…etc.) when an infrastructure-like project receives multiple sponsorships or has multiple partners that are separately funded.
  3. Instrument PI institution if none of the above applies.
  4. SMWG will continue to be the Naming Authority for Person resources.
